# How to Add a New Report via `report.add` Act

## Endpoint Overview
- **Act Name**: `report.add`
- **Access**: Authenticated users (Manager, Editor, Ordinary)
- **Authentication**: Required (JWT token in `Authorization: Bearer <token>` header)
- **Description**: Creates a new war crime report. The `reporter` relation is automatically set to the authenticated user. Supports optional relations to tags, category, documents, and geographic entities (hostile/attacked countries, provinces, cities).

---

## Request Structure
Request body follows the validator in `add.val.ts` with a top-level `details` object:
```typescript
{
  details: {
    set: { /* Pure fields + relation IDs */ },
    get: { /* Projection for response */ }
  }
}
```

---

### `set` Object (Required)
Combines **Pure Fields** (direct report attributes) and **Relation ID Fields** (references to related models).

#### Pure Fields (from `models/report.ts:report_pure`)
| Field | Type | Required | Default | Description | Example |
|-------|------|----------|---------|-------------|---------|
| `title` | string | ✅ | - | Report title | "Civilian Attack in Aleppo" |
| `description` | string | ✅ | - | Detailed incident description | "Airstrikes hit residential area..." |
| `location` | GeoJSON Point | ❌ | - | Incident location (GeoJSON format) | `{ type: "Point", coordinates: [36.2021, 36.2080] }` (lng, lat) |
| `address` | string | ❌ | - | Text address | "Salah al-Din District, Aleppo" |
| `status` | enum | ❌ | "Pending" | `Pending` \| `Approved` \| `Rejected` \| `InReview` | "Pending" |
| `priority` | enum | ❌ | - | `Low` \| `Medium` \| `High` | "High" |
| `selected_language` | LanguageCode | ✅ | - | Report language (e.g., "en", "fa", "ar") | "en" |
| `crime_occurred_at` | ISO Date String | ✅ | - | When crime occurred | "2024-05-03T14:30:00Z" |

*Note: `createdAt`/`updatedAt` are auto-generated by the backend - do not include them.*

#### Relation ID Fields (All Optional)
All IDs must be valid 24-character hex MongoDB ObjectIds. Omit if no relation needed.

| Field | Type | Description | Example |
|-------|------|-------------|---------|
| `tags` | string[] | Tag ObjectIds | `["507f1f77bcf86cd799439011"]` |
| `category` | string | Single Category ObjectId | `"507f1f77bcf86cd799439013"` |
| `documentIds` | string[] | Document ObjectIds | `["507f1f77bcf86cd799439014"]` |
| `hostileCountryIds` | string[] | Attacker/hostile country IDs | `["507f1f77bcf86cd799439015"]` |
| `attackedCountryIds` | string[] | Victim country IDs | `["507f1f77bcf86cd799439016"]` |
| `attackedProvinceIds` | string[] | Victim province IDs | `["507f1f77bcf86cd799439017"]` |
| `attackedCityIds` | string[] | Victim city IDs | `["507f1f77bcf86cd799439018"]` |

*Note: `reporter` is auto-set to the authenticated user - no need to send a reporter ID.*

---

### `get` Object (Required)
Projection specifying response fields. Use `selectStruct("report", 1)` for depth 1 (no relations) or specify fields directly:
```typescript
{ _id: 1, title: 1, status: 1, createdAt: 1 }
```

---

## Sample Request
```typescript
{
  details: {
    set: {
      title: "Civilian Attack in Aleppo",
      description: "Airstrikes hit residential area on May 3rd, 2024.",
      location: { type: "Point", coordinates: [36.2021, 36.2080] },
      address: "Salah al-Din District, Aleppo",
      priority: "High",
      selected_language: "en",
      crime_occurred_at: "2024-05-03T14:30:00Z",
      tags: ["507f1f77bcf86cd799439011"],
      hostileCountryIds: ["507f1f77bcf86cd799439015"],
      attackedCityIds: ["507f1f77bcf86cd799439018"]
    },
    get: { _id: 1, title: 1, status: 1, createdAt: 1 }
  }
}
```

---

## Response Structure
Standard Lesan format:
```typescript
{
  success: boolean,
  body: {
    _id: string, // New report ID
    title: string,
    status: string,
    // ... fields from `get` projection
  }
}
```

---

## Key Notes
1. **Auth**: Include JWT in headers: `Authorization: Bearer <token>`
2. **ObjectIds**: All relation IDs must be 24-char hex strings
3. **GeoJSON**: `location` uses `[longitude, latitude]` order
4. **Dates**: `crime_occurred_at` must be ISO 8601 string
5. **Relations**: Can be added/modified later via `report.updateRelations`
6. **Language Codes**: Valid values from `language_enums` (e.g., "en", "fa", "ar", "tr", "ku")
